# STM32学习日志（2026-05-31）

## 阶段一：STM32F407旧开发板锁死问题排查

### 事件背景

准备重新开始学习STM32。

手里原来有一块STM32F407VET6开发板，之前很久没使用。

重新打开工程后发现程序无法正常下载。

随后与开发板商家确认，发现该开发板使用的是：

25MHz外部晶振

而之前工程使用的是：

8MHz晶振配置

导致系统时钟配置错误。

------

### 问题现象

烧录程序后：

- ST-Link无法连接
- J-Link无法连接
- DAPLink无法连接
- Keil下载失败
- SWD接口失去响应

表现为：

芯片被“锁死”

------

### 排查过程

最开始怀疑：

- ST-Link损坏
- 驱动异常
- SWD配置错误

随后逐项排查：

#### 1、检查Keil设置

确认：

- Debug器选择正确
- SWD模式
- 下载算法正确

问题依旧存在。

------

#### 2、更换调试器

依次测试：

- ST-Link
- J-Link
- DAPLink

结果：

全部无法连接。

排除调试器故障。

------

#### 3、检查晶振配置

与淘宝客服确认后发现：

开发板使用：

25MHz晶振

而工程使用：

8MHz配置

例如：

```c
sys_stm32_clock_init(336,8,2,7);
```

属于：

8MHz晶振方案

如果直接用于25MHz晶振板卡：

PLL配置错误

系统时钟异常

可能导致SWD接口失效。

------

### 尝试的解锁方案

尝试过：

#### Boot0启动模式

Boot0接高电平

进入系统Bootloader

尝试重新下载程序。

------

#### Connect Under Reset

开启：

Connect Under Reset

降低SWD频率

尝试重新连接。

------

#### 全片擦除

尝试：

Erase Full Chip

重新烧录。

------

### 最终结果

虽然理论上可以继续解锁。

但：

- 排查时间过长
- 多个调试器均无法正常恢复
- 学习进度被严重拖慢

因此决定：

暂时放弃旧板子。

后续有时间再研究解锁。

### 第一个LED驱动

LED连接：

PB2

时钟使能：

```c
__HAL_RCC_GPIOB_CLK_ENABLE();
```

GPIO配置：

```c
GPIO_MODE_OUTPUT_PP
```

速度：

```c
GPIO_SPEED_FREQ_LOW
```

------

### 第一次失败

LED不闪。

检查发现：

main函数中只有：

```c
HAL_Init();
```

缺少：

```c
sys_stm32_clock_init(...);
delay_init(...);
```

------

### 第二次失败

加入：

```c
sys_stm32_clock_init(336,8,2,7);
```

LED仍然不闪。

------

### 最终成功

加入：

```c
delay_init(168);
```

之后：

LED开始正常闪烁。

代码：

```c
while(1)
{
    LED(1);
    delay_ms(200);

    LED(0);
    delay_ms(100);
}
```

运行正常。

## 学到的新知识

### HAL_Init()

作用：

初始化HAL库。

并不负责配置168MHz主频。

------

### sys_stm32_clock_init()

作用：

配置PLL。

最终得到：

168MHz系统时钟。

计算过程：

8MHz

↓

8/8

↓

1MHz

↓

1×336

↓

336MHz

↓

336/2

↓

168MHz

------

### delay_init()

作用：

告诉延时模块：

当前CPU运行频率是多少。

例如：

```c
delay_init(168);
```

表示：

CPU运行在168MHz。

否则：

delay_ms()

无法正常工作。
